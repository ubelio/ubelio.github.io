SC-401 Lab 4 – Configure Microsoft Purview Message Encryption

Enable and validate message encryption policies to protect sensitive email content across Microsoft 365.

Home Projects Scripts Home-Lab Config Labs Contact

Tools Used

  • Microsoft Learn Sandbox
  • Microsoft Purview Portal
  • Microsoft 365 Compliance Center
  • Outlook Web App

Steps Taken

  1. Signed into Microsoft Purview portal as MOD Administrator.
  2. Navigated to Information Protection → Message Encryption.
  3. Reviewed default encryption templates and enabled Microsoft Purview Message Encryption.
  4. Created a mail flow rule to apply encryption based on subject line keywords.
  5. Sent a test email from Outlook Web App with trigger keyword in subject.
  6. Verified encryption applied and recipient experience.

Outcome

Message encryption successfully configured and validated. Sensitive emails are now automatically encrypted based on policy triggers, supporting secure communication and compliance requirements.
