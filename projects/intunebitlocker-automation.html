Cloud and Co-Managed Device Encryption with Microsoft Intune

Challenge

Encrypt cloud-native and co-managed Windows devices using Microsoft Intune, ensuring compliance and recovery key protection without user disruption. The solution needed to support both Entra-joined and hybrid-joined devices enrolled via Intune or SCCM co-management.

Tools & Technologies

  • Microsoft Intune (Endpoint Security)
  • Microsoft Entra Security Groups
  • Co-management with Configuration Manager (SCCM)

Implementation & Environment

Depending on the device type, we used a Disk Encryption policy from Intuneâ€™s Endpoint Security tab. This policy functions similarly to an ADMX-backed GPO, defining encryption methods and enforcement for OS, fixed, and removable drives.

Steps:

  1. Create a Disk Encryption policy in Intune.
  2. Assign the policy to a Microsoft Entra device group.
  3. Cloud-only devices receive and apply the policy automatically.
  4. For hybrid-joined or co-managed devices:
    • Ensure the device is enrolled in Intune or co-managed via SCCM.
    • Switch the Endpoint Protection workload to Intune in SCCM to allow BitLocker settings to apply.
