New User Creation Automation

PowerShell-Driven Onboarding Workflow

Challenge

Helpdesk technicians were spending 30+ minutes manually creating new users, assigning groups, and provisioning mailboxes. The process was error-prone and inconsistent across departments.

Tools & Technologies

  • PowerShell
  • Active Directory (on-prem)
  • Exchange (on-prem + cloud)
  • Group-based licensing
  • OU and attribute-based provisioning

Implementation

  1. Script prompts helpdesk tech for:
    • First name, last name
    • Department and supervisor
    • Licensing group
  2. Creates AD user with proper OU placement and attributes
  3. Adds user to department and licensing groups
  4. Triggers mailbox creation (on-prem and cloud)
  5. Logs all actions for audit and rollback

# Prompt for user details
$FirstName = Read-Host "Enter First Name"
$LastName = Read-Host "Enter Last Name"
$Department = Read-Host "Enter Department"
$Manager = Read-Host "Enter Manager's Username"
$LicenseGroup = Read-Host "Enter Licensing Group"

# Construct username and email
$Username = "$FirstName.$LastName"
$Email = "$Username@yourdomain.com"

# Create AD user
New-ADUser -Name "$FirstName $LastName" -SamAccountName $Username `
  -UserPrincipalName $Email -Department $Department `
  -Manager $Manager -Path "OU=$Department,DC=yourdomain,DC=com" `
  -AccountPassword (ConvertTo-SecureString "TempP@ss123" -AsPlainText -Force) `
  -Enabled $true

# Add to groups
Add-ADGroupMember -Identity $LicenseGroup -Members $Username
Add-ADGroupMember -Identity "Dept-$Department" -Members $Username

# Mailbox provisioning
Enable-Mailbox -Identity $Username
Enable-RemoteMailbox -Identity $Username -RemoteRoutingAddress "$Username@tenant.mail.onmicrosoft.com"

Write-Host "User $Username created and provisioned successfully."

Impact

  • Saved ~30 minutes per user onboarding
  • Eliminated manual errors and skipped steps
  • Improved consistency across departments
  • Enabled scalable, role-based provisioning
