Challenge

Microsoft Entra Connect was running on a single server, creating a risk of sync failure during outages or upgrades. The goal was to deploy a staging server to ensure high availability and enable seamless disaster recovery.

Tools & Technologies

  • Microsoft Entra Connect
  • Staging Mode Configuration
  • Synchronization Service Manager
  • Microsoft Entra Connect Health
  • Disaster Recovery Planning

Implementation

  2. Provisioned a second server and installed Microsoft Entra Connect in staging mode
  3. Configured sync scheduler and verified full import and delta sync operations
  4. Ensured configuration parity between active and staging servers
  5. Validated export preview and failover readiness using CSAnalyzer and export logs
  6. Documented failover procedures and tested switch to active mode
