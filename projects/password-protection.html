Challenge

Weak and guessable passwords posed a risk across both cloud and on-prem environments. The goal was to enforce strong password hygiene, block company-related terms, and align lockout behavior across Microsoft Entra and Active Directory.

Tools & Technologies

  • Microsoft Entra Password Protection
  • Entra Connect Server Policies
  • On-Prem AD Password Protection Agents
  • Group Policy Objects (GPOs)
  • Cloud Lockout Thresholds

Implementation

  1. Installed password protection agents on all domain controllers
  2. Configured banned password list to block company-related and weak terms
  3. Set password protection policies on Entra Connect server
  4. Applied GPOs to enforce hybrid password requirements
  5. Enabled cloud-based lockout policies that apply to on-prem accounts
  6. Validated enforcement across both environments with test accounts and audit logs
